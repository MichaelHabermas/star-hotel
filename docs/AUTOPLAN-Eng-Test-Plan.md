## Key interactions to verify

- Create reservation: valid dates, overlapping room rejection, totals match extracted `.bas` logic.
- Edit + delete: optimistic vs confirmed behavior once specified.
- IPC failure: renderer shows normalized error, no shell crash.
- DB busy: user-visible retry or message (not white screen).

## Edge cases

- Leap-year and same-day check-in/out boundaries.
- Partial-day billing if legacy supports it (confirm against `modLogic.bas` when wired).
- Malformed migration rows: import rejects or quarantines with operator-visible summary.
- Two windows / rapid clicks: single-writer SQLite policy respected.
