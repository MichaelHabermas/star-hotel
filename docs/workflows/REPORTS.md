# Reports Workflow

## Purpose

Reports is for printing and operational review after front-desk data is already in place.

## Intended usage

1. Finish the transaction first in Booking, Customer, or Room Board.
2. Open Reports from `F2` or navigation.
3. Run the needed report:
   - day sheet
   - folio or receipt
   - other print-oriented outputs

## Best practice

- use reports after saving the booking state
- use reports to verify or print, not to drive primary room operations
- return to Room Board after printing so the live room picture stays central
