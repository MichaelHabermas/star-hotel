# Customer Workflow

## Purpose

Customer is for finding people and reviewing their prior booking activity.

## Intended usage

1. Open Customer from `F3` or the navigation.
2. Search by one or more filters:
   - name
   - passport or ID
   - origin
   - contact
   - booking date range
3. Review the customer results list.
4. Select a customer.
5. Review the booking history for that customer.

## When to use it

- guest walks in and you need to check whether they stayed before
- guest calls and you need to pull up prior history
- you need to confirm identity or contact details before a booking update
- you need historical transaction context before printing or reporting

## Intended mental model

Customer is a search-and-history workspace, not the primary place to start a new stay. Use it to identify the person, then move back into Booking or Room Board to continue the transaction.
