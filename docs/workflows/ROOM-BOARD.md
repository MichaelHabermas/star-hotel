# Room Board Workflow

## Purpose

The Room Board is the main command surface. It is where the operator keeps the live picture of the hotel and branches into booking or room work.

## What you should see

- status summary counts
- fixed board layout by level
- color-coded room states
- a compact room menu for the selected room
- overflow room ledger only for rooms outside the fixed legacy board

## How to use it

1. Scan the summary counts.
2. Select a room on the board.
3. Use `Enter` or click to open booking for that room.
4. Use the Room Menu to:
   - open booking
   - edit room
   - change simple room statuses like housekeeping or maintenance when appropriate

## Keyboard

- `Esc`: go to Room Board
- `Arrow keys`: move around the fixed board
- `Enter`: open booking for the active room
- `F1`: Booking
- `F2`: Report
- `F3`: Customer
- `F4`: Room

## Status meaning

- Open: room is available for booking
- Booked: room has a future or current booking not yet checked in
- Occupied: guest is in house
- Housekeeping: room is out of service for cleaning turnover
- Maintenance: room is blocked for repair or setup

## Intended operator pattern

Stay on the Room Board as much as possible. Open other screens to finish a task, then return so the board remains your live operational map.
