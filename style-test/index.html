Style Lab • Reservations / Check-In

Three directions for the screen the front desk will live in.

Each concept uses the same operational surface: room status, guest and stay details, pricing, recovery states, and arrival activity. That keeps the test focused on tone, density, hierarchy, and polish rather than feature drift.

Anchor screen Reservations and check-in, ranked highest in the docs.
Active directions Lakeside Console for light mode, Night Audit for dark mode.
What stays fixed Keyboard-first layout, summary rail, room-status companion.

Why this screen

The project docs consistently put reservations / check-in first, with rooms management second. It is the best single canvas for deciding the app’s visual language because it combines dense data, operational urgency, and hospitality tone.

The chosen split is now clear: light mode should feel like a fast workstation, and dark mode should feel deliberate rather than simply inverted.
